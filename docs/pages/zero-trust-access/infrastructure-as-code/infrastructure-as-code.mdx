>
  There are two ways to configure a Teleport cluster:
 
  - **Static configuration** sets up how Teleport runs, such as enabled services and ports. It is read at startup  (the default path is `/etc/teleport.yaml`).
  - **Dynamic resources** define access and protected infrastructure, such as users, roles, and clusters. They are stored by the Auth Service and can be updated anytime using the CLI or infrastructure-as-code tools.
</DocHero>
